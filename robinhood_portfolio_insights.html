What is Robinhood?

Robinhood is a desktop and mobile trading platform for the everyday person to invest in stocks, exchange-traded funds, cryptocurrency, and more, all commission free. When Robinhood Markets, Inc. was founded in 2013, it set out to make a platform for the modern day investor to make commission free trades, day after day. Robinhood has become incredibly popular and has over 15 million people using it as their primary trading platform (2022).

I use Robinhood as a platform to invest in exchange-traded funds (ETFs) for an easy way to diversify my portfolio while directing my investments towards are particular sector or group of companies. Read more about ETFs on Investopedia.com here.

What's lacking?

Robinhood’s platform has always been very sleek and user-friendly. However, there is a problem for user’s that are primarily invested in ETFs. The banner image at the top of this page shows what is supposed to be high-level insight into a user’s portfolio. The problem is, if you’ve sought out to use Robinhood to solely invest in ETFs, you already know that your profiles investments are 100% distributed in them. What could be a very useful pie chart, is lacking key information for the account.

Project Description

Using the robin_stocks module and an encrypted file with login information, a user's Robinhood account can be accessed inside of the Notebook. If the login.txt file is encrypted at the time of use, or the incorrect path is specified, a picture is taken through the webcam and saved with the timestamp and file path attempted as the file name. Once accessed, the portfolio's ETF tickers are put into various webscraping functions. The first function scrapes the sector breakdown data for each ETF and writes that information into a dictionary, where it is then converted into a Pandas Dataframe. The second webscraping function retrieves each ETF's top stock holdings and calculates the percentage of the whole portfolio that each stock takes up. By calculating the true percentage prior to dictionary assignment, if the same stocks are found again, their percentage totals are able to be accurately added together. Once these calculations are made, the data is stored into a dictionary, and once again converted into a DataFrame. Now that there is a list of the top N stocks 'held' in the portfolio, each stock needs to be assigned a sector so that the sector and stock data can be joined. There is a fourth working function that is not currently in use: it uses the InvestPy module to compile a 10 year market history for each stock in the top stocks list. The InvestPy module was not consistantly retrieving the stock data and the function needs more work to make it dependable. After all of this data has been stored as DataFrames, it is written to a Google Sheet via the Google Sheet API, an API key, and sheet information. This sheet serves as the live server for the Tableau Dashboard. Lastly, in Tableau, the sector breakdown of the ETF portfolio and top holdings of the ETFs can be seen. After the InvestPy function is improved, a Tableau Dash providing 10 year market history data for each ETF and holdings can be added.
